Current Notes

This sandbox favors isolation and clarity over product polish.

  • The site boots with a fixed default model and exposes only a conservative bundled catalog of INT8 ONNX variants that fit the browser track.
  • If the selected model fails, the sandbox keeps the previous active model when possible and otherwise falls back to the same rough matte heuristic as the native fallback.
  • The source input accepts one video, one still image, or a naturally sorted image sequence. Webcam capture stays local through getUserMedia.
  • The stage preview now respects the loaded media aspect ratio while processing still uses discrete browser-safe resolution presets internally.
  • Process Full Media exports a browser-side result for still images, image sequences, or videos. Video export remains a WebM preview path in this sandbox, not a frame-exact mastering pipeline.
  • Recording captures the output canvas through MediaRecorder and saves a WebM preview.
  • For heavier exports, the next upgrade is replacing the canvas loop with WebCodecs decode and encode.
